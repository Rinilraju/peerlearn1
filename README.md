# PeerLearn Collaboration Platform

Welcome to the PeerLearn project! This repository contains both the frontend (React/Vite) and backend (Node.js/Express) code.

## Prerequisites

Before you start, make sure you have the following installed on your machine:
*   [Node.js](https://nodejs.org/) (v16 or higher)
*   [PostgreSQL](https://www.postgresql.org/download/)

## Setup Instructions

### 1. Clone the Repository
```bash
git clone <your-github-repo-url>
cd PeerLearn
```

### 2. Frontend Setup
1.  Navigate to the root directory (where `package.json` for frontend is located).
2.  Install dependencies:
    ```bash
    npm install
    ```

### 3. Backend Setup
1.  Navigate to the `backend` directory:
    ```bash
    cd backend
    ```
2.  Install backend dependencies:
    ```bash
    npm install
    ```
3.  **Environment Variables:**
    Copy the `.env.example` file to create your own `.env` file.
    *(Note: The `.env` file is meant to stay on your local computer safely hidden from Github)*
    ```bash
    cp .env.example .env
    ```
    Open the `.env` file and update the `DB_PASSWORD` to your local PostgreSQL password. Also update `JWT_SECRET` for testing, and `EMAIL_USER`/`EMAIL_PASS` if you need to test the email verification system locally.

### 4. Database Setup
1.  Ensure your local PostgreSQL server is running.
2.  Inside the `backend` directory, run the database creation script:
    ```bash
    node create_db.js
    ```
    *(Note: if this fails, you may need to manually create a database named `peerlearn` using pgAdmin or the psql CLI).*
3.  Once the database is created, run the migrations script to configure the schema:
    ```bash
    node feature_migration.js
    ```

## Running the Application Locally

You will need to open **two** terminal windows.

**Terminal 1 (Frontend):**
Navigate to the root directory of the project and run:
```bash
npm run dev
```

**Terminal 2 (Backend):**
Navigate to the `backend` directory and run:
```bash
node src/server.js
```

The frontend will start (usually on `http://localhost:5173`) and connect to the backend API (`http://localhost:5000`).

## Deployment to Render

This project is configured for easy deployment on [Render](https://render.com/) via the `render.yaml` blueprint.

### Automatic Setup
1.  Push your code to GitHub.
2.  Connect your GitHub repository to Render.
3.  Render will automatically detect the `render.yaml` file and prompt you to create the following resources:
    *   **PostgreSQL Database**: Managed database for PeerLearn.
    *   **Backend Service**: Node.js Express API.
    *   **Frontend Site**: Static site for the React application.

### Important Configuration
After the initial deployment, you must manually set the following environment variables in the Render dashboard for the **peerlearn-backend** service:
- `EMAIL_USER`: Your Gmail address for notifications.
- `EMAIL_PASS`: Your Gmail App Password.
- `JWT_SECRET`: (Optional) Already auto-generated by the blueprint.

The blueprint automatically links the Frontend to the Backend using the `VITE_API_URL` environment variable.

## Tech Stack
*   **Frontend:** React, TypeScript, Tailwind CSS, Vite, WebRTC (`simple-peer`), Socket.io-client
*   **Backend:** Node.js, Express, PostgreSQL (`pg`), Socket.io, JWT Authentication
